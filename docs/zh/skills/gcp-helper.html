☁️ GCP 小幫手

透過 gcloud CLI 管理 GCE、GCS 或 GKE 等 GCP 資源。

概覽

藉由提供標準化的指令和最佳實踐，協助管理 Google Cloud Platform (GCP) 資源。此技能可幫助 LLM 代理安全地操作複雜的雲端環境。

使用場景

  • 管理 GCE 執行個體、GCS 儲存桶或 GKE 叢集。
  • 檢查 IAM 權限或設定服務帳戶。
  • 部署 Cloud Functions 或 Cloud Run 服務。

核心模式

在進行任何變更之前，請務必先確認目前啟動中的專案，以避免誤動到正式環境。

# 確認專案
gcloud config get-value project

快速參考

服務 常用指令
GCE gcloud compute instances list
GCS gsutil ls
GKE gcloud container clusters list

實作步驟

  1. 確認目前專案：gcloud config get-value project。
  2. 列出特定服務的資源：gcloud [service] list。
  3. 使用已確認的參數執行操作。

常見錯誤

  • 專案錯誤： 未經確認就在錯誤的專案中執行操作。
  • 未驗證： 忘記登入驗證 (gcloud auth login)。
  • 權限不足： 在沒有必要 IAM 角色的情況下執行指令。
